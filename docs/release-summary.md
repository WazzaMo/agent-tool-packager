# Release summary instruction

In docs/release-notes please create release-summary-{version}.md for each release note that has a summary paragraph for all level 1, 2 and 3 heading sections. Don't just describe
the release note content in the summary but try to convey its meaning so that
if the original release note has a section with a list of entries and improvements,
summarise the meaning of those improvements and give a brief list, in a sentence, so
that the reader still gets the idea how many things were improved but list only the first ten
items with "and so on (see original release note in Github)" after that.
