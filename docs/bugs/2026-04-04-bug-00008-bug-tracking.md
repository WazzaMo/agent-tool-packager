# Bug Tracking

This bug document describes how bug tracking will be done.
It mirrors [Issue #8](https://github.com/WazzaMo/agent-tool-packager/issues/8)

## Description

Bugs are tracked in development using a bug document with a date and a bug ID.

It is expected that bugs will be:

1.  registered here as an issue;
2.  if accepted or not responded to here as first communication;
3.  reviewed and added as a bug document;
4.  root cause analysed
5.  fixed.
6.  The Github issue should be responded.

This issue will remain as a form of documentation.

## Result

No change. This document reflects Issue #8 in that it provides documentation for the
bug tracking process and creates a template of sorts.
